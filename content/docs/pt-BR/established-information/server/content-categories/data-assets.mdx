# Recursos de Dados (Data Assets)

Data Assets são arquivos JSON que controlam o comportamento de gameplay e conteúdo principal do Hytale. Eles proporcionam uma abordagem poderosa e orientada a dados para a criação e modificação de elementos de jogo sem a necessidade de escrever código.

## O que são Data Assets?

Data Assets permitem você a criar ou modificar elementos fundamentais do jogo através de arquivos de configuração JSON:

* **Blocos**: Defina novos tipos de blocos e suas propriedades
* **Itens**: Crie itens personalizados com comportamentos únicos
* **NPCs**: Configurar personagens não-jogador e sua IA
* **Geração de mundo**: Controle como o mundo é gerado
* \*\*Tabelas de Loot \*\*: Defina quais itens são dropados de várias fontes

## Comparação com Minecraft

Data Assets são análogos aos arquivos JSON de data packs do Minecraft, mas com um escopo significativamente mais amplo. Enquanto os data packs do Minecraft são um pouco limitados, a arquitetura orientada a dados do Hytale significa que elementos principais do jogo como blocos, itens, NPCs podem todos ser modificados via arquivos JSON.

Esta abordagem expandida baseada em dados dá aos modders muito mais poder para personalizar o jogo sem precisar mergulhar em programação complexa.

## Ferramentas de Desenvolvimento

Reconhecendo que a edição de JSON pode ser tediosa e propicia a erros quando feito à mão, Hytale pleneja fornecer várias ferramentas para tornar mais acessível o trabalho com os Data Assets:

### Editor de Nós

Hytale está desenvolvendo um "Editor de Nós" para fazer o gerenciamento de Data Assets mais amigável:

* **Interface visual**: Fornece uma maneira gráfica para trabalhar com o relacionamentos dos dados
* **Foco em customização**: particularmente útil para customizar NPCs, interações e geração do mundo
* **Fluxo de trabalho integrado**: Funciona diretamente no Hytale Asset Editor para um conjunto unificado de criação
* **Acessibilidade**: facilita a compreensão e modificação de relações complexas de dados

### Ferramentas Adicionais

Além do Editor de Nó, o Hytale planeja suportar:

* **Geradores JSON Visuais**: Interfaces gráficas para criar JSON sem escrever sintaxe
* **Editores baseados em formatos**: Formatos com estruturas que geram o JSON correto automaticamente
* **Diversas opções de fluxos de trabalhos**: Ferramentas diferentes para diferentes níveis de conforto, e para usos específicos

## Capacidade e Simplicidade

A eficácia dos Data Assets são demonstradas pela sua simplicidade. Por exemplo, tornar raptors montáveis em Hytale requer somente 4 linhas de configurações JSON - mostrando quão capaz e acessível o sistema projetado para ser.

Esse equilíbrio entre capacidade e acessibilidade é o cerne da filosofia de modding do Hytale: oferecendo ferramentas de nível profissional que não necessita de conhecimento em programação para usa-las.

## Integração com Outros Sistemas

Data Assets funcionam perfeitamente com outras ferramentas de desenvolvimento para o Hytale:

* **Visual Scripting**: Pode referenciar e manipular dados específicos nos Data Assets
* **Java plugins**: Pode ler e modificar Data Assets para comportamentos mais complexos
* **Asset Editor**: Oferece uma interface única para gerenciar assets visuais e suas configurações de dados

Essa integração garante que os Data Assets não sejam arquivos isolados, mas que façam parte do sistema amplo que é o desenvolvimento do Hytale.
