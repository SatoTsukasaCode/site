# Вносить вклад в HytaleModding

Мы приветствуем вклад сообщества для улучшения и расширения документации для моддинга. Любой Ваш вклад значим для нашего сообщества: Исправление опечаток, добавление руководств, примеры кодов

## Как внестис свой вклад

### Сообщить о проблеме

Если вы нашли проблему с документацией, то, пожалуйста, сообщите об этом, открыв новую проблему в нашем [GitHub репозитории](https://github.com/HytaleModding/site/issues).

Если вы можете работать с проблемой, то вас могут попросить быть назначенным на это и подтвердить PR

### Отправка Pull Request'ов

1. Форкнуть репозиторий на GitHub.
2. Клонируйте свой форк на вашу локальную машину.
3. Создать новую ветку для ваших изменений

```bash
git checkout -b my-feature-branch
```

4. Внесите свои изменения и оставьте достаточно ясный комментарий по поводу самого  Мы следуем [Обычным commit](https://www.conventionalcommits.org/en/v1.0.0/), ваш PR не будет принят, если ваши сообщения для commit не будут следовать этим условиям.

```bash
git commit -m "Добавьте фичу Х в такую-то документацию"
```

5. Внесите ваши изменения в ваш форк

```bash
git push origin my-feature-branch
```

6. Откройте пул запроса на оригинальном репозитории, опишите причину по которым Вы решили внести данные изменения.
