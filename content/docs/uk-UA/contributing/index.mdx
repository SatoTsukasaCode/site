# Внесок до HytaleModding

Ми вітаємо внески від спільноти, які допоможуть поліпшити та розширити документацію HytaleModding. Незалежно від того, чи ви виправляєте помилки, додаєте нові підручники чи надаєте приклади коду, ваш внесок є цінним!

## Як зробити свій внесок

### Повідомити про проблему

Якщо ви виявили проблему з документацією, будь ласка, повідомте про неї, створивши нову проблему в нашому [репозиторії GitHub](https://github.com/HytaleModding/site/issues).

Якщо ви можете самостійно вирішити цю проблему, можна попросити, щоби вам її призначили, та надіслати запит на злиття.

### Надсилання запитів на злиття

1. Створіть гілку репозиторію на GitHub.
2. Склонуйте свою гілку на свій локальний комп'ютер.
3. Створіть нову гілку для своїх змін.

```bash
git checkout -b my-feature-branch
```

4. Внесіть свої зміни та зафіксуйте їх із чіткими повідомленнями. Ми дотримуємося [загальноприйнятих правил фіксації](https://www.conventionalcommits.org/en/v1.0.0/), і ваш запит на злиття не буде прийнятий, якщо ваші повідомлення про фіксацію не відповідають цим правилам.

```bash
git commit -m "Додати особливість X до документації"
```

5. Відправте свої зміни до своєї гілки.

```bash
git push origin my-feature-branch
```

6. Відкрийте запит на злиття в оригінальному репозиторії, описавши свої зміни та причини, чому їх слід об'єднати.
