# Segíts a HytaleModdingnak!

Mind a HytaleModding dokumentációjának javításában, mind pedig annak kiegészítésében szívesen fogadunk segítséget a közösségtől. Nem számít, hogy csak egy elírást javítanál, vagy esetleg új útmutatót vagy kódpéldákat adnál hozzá - a hozzájárulásod értékes számunkra!

## Hozzájárulás menete

### Hiba bejelentése

Ha valamilyen hibát találsz a dokumentációban, akkor kérlek a [GitHub _repository_](https://github.com/HytaleModding/site/issues)-ban nyiss neki egy új _issue_-t, amin keresztül jelezheted azt.

Ha magadtól is ki tudod javítani, akkor jelezd nekünk a javítási szándékot és nyújts be egy PR-t (_Pull Request_) GitHubon.

### _Pull Request_ek benyújtása

1. Másold le a GitHub _repository_-t (készíts egy _fork_ot).
2. A másolatot mentsd le a saját eszközödre.
3. Készíts egy új _branch_-et a változtatásaidnak. (Angol legyen a _branch_ neve és a _commit_ üzenetek is.)

```bash
git checkout -b my-feature-branch
```

4. Készítsd el a változtatásaid és adj hozzá érthető _commit_ üzenetet.  Megadott [konvenciókat](https://www.conventionalcommits.org/en/v1.0.0/) követünk a _commit_ üzenetek esetén. A te PR-odnak is ezeket kell követnie, különben nem lesz elfogadva.

```bash
git commit -m "Add feature X to documentation"
```

5. _Push_old a változtatásaid a másolatodra.

```bash
git push origin my-feature-branch
```

6. Nyújts be egy PR-t az eredeti _repository_-n, ahol leírod mit változtattál és miért gondolod elfogadhatónak.
