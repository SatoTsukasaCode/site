# Внесок до HytaleModding

Ми вітаємо внески від спільноти, щоб допомогти покращити і розширити документацію HytaleModing. Незалежно від того, чи ви виправляєте помилку, додаєте нові підручники, або ж надаєте приклади коду, ваші внески цінні!

## Як допомогти

### Повідомити про проблему

Якщо ви виявили проблему з документацією, будь ласка, повідомте про неї, створивши нову проблему в нашому [репозиторії GitHub](https://github.com/HytaleModding/site/issues).

Якщо ви можете її виправити самостійно, ви можете запитати про призначення її та відправити PR.

### Надсилання запитів на злиття

1. Розгалужте репозиторій на GitHub.
2. Клонуйте вашу вилку до локальної машини.
3. Створіть нову гілку для ваших змін.

```bash
git checkout -b my-feature-branch
```

4. Внесіть зміни та закомітьте їх чіткими повідомленнями. Ми слідуємо [звичайним комітам] (https://www.conventionalcommits.org/en/v1.0.0/), ваш запит на злиття не буде прийнято, якщо повідомлення комітів не будуть дотримуватися цієї умови.

```bash
git commit -m "Додати функцію X до документації"
```

5. Відправити зміни до вашого форку.

```bash
git push origin my-feature-branch
```

6. Відкрийте запит на злиття в оригінальному репозиторії, описуючи ваші зміни й чому їх слід об'єднати.
